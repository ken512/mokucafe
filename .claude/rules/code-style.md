# コーディングルール

- コンポーネントは関数コンポーネント（アロー関数）で統一
- any型の使用禁止
- 単一原則に沿ってコンポーネントを作成
- フォームは react-hook-form を使用
- コメントは日本語で記述
- 変数名・関数名はキャメルケース（camelCase）
- Reactコンポーネントファイル名はPascalCaseで始める
- TypeScriptファイル名はキャメルケース（camelCase）で始める

## コンポーネントの関心分離

| 観点 | NG（Before） | OK（After） |
|------|-------------|------------|
| データ取得 | UIコンポーネントが持つ | hooksに分離する |
| 削除・更新処理 | UIコンポーネントで持つ | 責務に応じてhooksに配置 |
| 整合性担保 | UIが担保 | hooksが担保 |
| UIの責務 | ロジック混在 | 表示に集中 |
| 再利用性 | ほぼ不可能 | hooksで使い回せる |
| UIの純粋性 | 外部通信に依存 | propsベースでシンプル |

- データ取得・副作用・状態管理は `hooks/` に切り出す
- UIコンポーネントはpropsを受け取って表示するだけにする
- ロジックを持つコンポーネントと表示専用コンポーネントを混在させない

## 汎用コンポーネントの設計

- 複数箇所で使うUIは `src/components/ui/` に汎用コンポーネントとして切り出す
- propsで挙動・スタイルを制御できるよう設計し、記述量を減らす
- 同じようなJSXを複数箇所に書かない（DRY原則）
- 汎用コンポーネントはロジックを持たず、表示のみに責務を限定する
